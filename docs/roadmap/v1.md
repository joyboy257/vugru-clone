# Roadmap — v1

Target: working end-to-end prototype that a real estate agent can use.

## What's built

### Auth
- [x] Magic link login (`POST /api/auth/login` → issues JWT)
- [x] Signup (`POST /api/auth/signup` → creates account with 1000 free credits)
- [x] JWT middleware on protected routes
- [x] Logout (`POST /api/auth/logout`)
- [x] **Password reset** (`POST /api/auth/forgot-password` + `POST /api/auth/reset-password`)
- [x] **Google OAuth** (`GET /api/auth/google` + `GET /api/auth/google/callback`)
- [x] **Email + password login** (password field added to `/api/auth/login` and `/api/auth/signup`)
- [x] **Magic link rate limiting** (Redis sliding window via `lib/rate-limit.ts`)
- [x] **Password change** (`PATCH /api/auth/password` for logged-in users)

### Core loop
- [x] Project CRUD (`/api/projects`)
- [x] Photo upload (presigned R2 URL → `POST /api/upload/confirm`)
- [x] Photo delete + reorder (`DELETE /api/photos/[id]`, `POST /api/projects/[id]/reorder`)
- [x] Clip generation request + queue (`POST /api/clips/generate` → BullMQ)
- [x] Clip status polling (`GET /api/clips/[id]`)
- [x] Clip download (`GET /api/clips/[id]/download`)
- [x] GPU worker: Runway Gen-3 / SVD → R2 pipeline (`gpu-worker/`)
- [x] Credit deduction on clip generate (deducts before queue)
- [x] **Batch clip generation** (`POST /api/projects/[id]/clips/batch-generate` → 202, queues all photos)
- [x] **Resolution selector UI** — pill toggle in batch modal, dynamic credit cost display

### Virtual staging
- [x] Worker (`workers/virtual-stage/`)
- [x] API route (`POST /api/photos/[id]/virtual-stage`)
- [x] Editor UI (style picker, before/after)

### Sky replacement (NEW — sprint 2026-03-27)
- [x] `POST /api/photos/[id]/sky-replace` route — auth, credit check, enqueue
- [x] `workers/sky-replace/` worker — RMBG-1.4 segmentation + sharp compositing
- [x] Schema: `skyStyle`, `skyStorageKey`, `skyReplacedAt` columns on photos

### Auto-edit (NEW — sprint 2026-03-27)
- [x] `POST /api/auto-edits/[id]/render` route — validates, deducts credit, enqueues
- [x] `workers/auto-edit-render/` worker — FFmpeg concat + title overlay + music mix
- [x] `lib/music.ts` — 5 ambient music tracks (upbeat-1, warm-1, modern-1, cinematic-1, acoustic-1)
- [x] Generate Video UI in project editor — modal, music picker, title input, video preview

### Billing
- [x] Stripe checkout (`POST /api/billing/checkout`)
- [x] Stripe webhook handler (`POST /api/webhooks/stripe`)
- [x] Credit balance fetch (`GET /api/billing/credits`)
- [x] Credit history (`GET /api/billing/history`)
- [x] Billing frontend (`/dashboard/billing`)

### Frontend
- [x] Landing page (hero, how-it-works, features, pricing)
- [x] Magic link login + signup pages
- [x] Dashboard (project list, stats)
- [x] Project editor (photo grid with reorder/delete, clip generation with status polling)
- [x] Settings page

## Architecture discoveries (sprint 2)

| Finding | Detail |
|---------|--------|
| **CogVideoX is legacy** | `replicate.ts` in `gpu-worker/` and `workers/video-render/` is dead code. Active pipeline: Runway Gen-3 Alpha Turbo (`gen3a_turbo`) + Stable Video Diffusion |
| **Two gpu-worker directories** | `gpu-worker/` (Next.js companion, handles clip generation via `clipProcessor.ts`) and `workers/video-render/` (standalone worker). Only `gpu-worker/` is the active pipeline |
| **REPLICATE_API_TOKEN** | Not needed — Runway/SVD are the active providers |

## Blockers

1. **R2 / Cloudflare credentials** — `.env.local` has them commented out
2. **Stripe keys** — commented out in `.env.local`
3. **Resend API key** — for password reset and clip-complete emails
4. **Google OAuth credentials** — for production OAuth flow
